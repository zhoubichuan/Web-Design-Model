### 设计原则

### solid 五大设计原则

* 单一职责原则
  * 一个程序只做好一件事
  * 如果功能特别复杂就进行拆分

- 开放封闭原则

  * 对扩展开放，对修改关闭
  * 增加需求是，扩展新代码，而非修改已有代码

- 里氏替换原则

  * 子类能覆盖父类
  * 子类能出现的地方子类就能出现

- 接口隔离原则

  * 保持接口的单一独立，避免出现胖接口

- 依赖倒置原则
  * 面向接口编程，依赖于抽象而不是依赖于具体实现
  * 使用方只关注接口而不关注具体的实现
